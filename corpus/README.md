# corpus/

Committed test files only. A file may live here only if it is **self-generated**
(our own exports from PowerPoint, Excel, LibreOffice Draw, etc.) or
**verifiably CC0/public domain**. Everything else — GPL-sourced test resources,
work-sourced files — belongs in the gitignored `corpus-local/` and is never
committed, screenshotted, or quoted in issues, commits, or docs.

## Manifest

Every `.emf` here was generated by us: the `.svg` next to it is the authored
source, converted with LibreOffice headless (`soffice --convert-to emf`,
LibreOffice 26.2.4.2 on macOS). LibreOffice's exporter wraps output in a
vestigial dual-mode EMF+ comment shell (header + two rendering hints + EOF,
no EMF+ drawing) and under-reports the header `Records` field by one — both
quirks are intentional test surface.

| File | Phase | Drawing records | Notes |
|---|---|---|---|
| gate-p2-star | 2 | EMR_POLYGON | ten-point star + zigzag (strokes tessellated to polygons by the exporter) |
| gate-p2-house | 2 | EMR_POLYGON | house scene, 7 filled/stroked polygons incl. 16-gon sun |
| gate-p2-triangles | 2 | EMR_POLYGON | overlapping triangles + polyline W |
| gate-p3-shapes | 3 | paths: FILLPATH, LINETO, POLYBEZIERTO | rects/ellipse/line exported via path brackets |
| gate-p3-curves | 3 | paths: FILLPATH, POLYBEZIERTO | cubic béziers + rounded rect via path brackets |

All five also carry the standard LibreOffice state preamble (map mode,
window/viewport, SAVEDC/RESTOREDC, SETROP2, one full-canvas
EMR_INTERSECTCLIPRECT) — phase-2 renderable except where marked phase 3.
